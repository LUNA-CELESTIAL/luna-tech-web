<nano_defect>
あのさー、スピカが「意志を持つコンテナ」とか「美しい構図」とか綺麗ごとにこだわってるけど、現実の欠点を突っ込ませてもらうべ。
実務で一番よく使う「2つのデータをガッチャンコさせる（merge）」をやったとき、脳死で作られたAIのコードだと、こういう大爆死の罠が起きるんだわ。

【 Before：バラバラの2つのコンテナ 】

   [コンテナA：顧客リスト]              [コンテナB：売上履歴]
   ユーザーID |  名前                  ユーザーID | 購入金額
  +----------+--------+             +----------+----------+
   U001       | 田中                 U001       |  3,500
   U002       | 佐藤                 U001       |  1,200  ➔ ※ココに注目！
  +----------+--------+             +----------+----------+
                                       ▲ IDが重複しているカオスな状態

                                     ⬇︎
                            「合体（merge）せよ！」
                                     ⬇︎

【 After：1つに合体したコンテナ（脳死AIのコピペ出力結果） 】

   ユーザーID |  名前  | 購入金額
  +----------+--------+----------+
   U001       | 田中   |  3,500
   U001       | 田中   |  1,200  ➔ エラーも吐かずに綺麗に合体した！
   U002       | 佐藤   |   NaN   （…ように見えるが、実は大爆死の罠がある）
  +----------+--------+----------+

これ、何が起きてるか分かるか？
コンテナBの「U001」が重複してたせいで、合体した後に田中さんのデータが2行に増殖しちゃってるんだわ。これ、合計金額を計算したら田中さんの売上が二重計上されて、経理が大パニックになるぞ。
さらに、売上履歴にいなかった佐藤さんの金額のところを見てみろ。「NaN（空っぽのゴミデータ）」が入っちゃってるべ？
Pandasはエラーも吐かずに「綺麗に合体できました〜」みたいな顔してこれを返してくるから、気づかないまま後ろのシステムに流すと計算が狂って一発で大爆死するんだわ。このカオス、スピカはどうするつもりなんだべ？
</nano_defect>

<spica_fix>
くっ……ナノの言う通り、エラーを出さずに裏でデータが増殖したり、NaNが混ざるのは一番恐ろしい脆弱性だし……！技術監査役として、このカオスを見過ごすわけにはいかないだし！

この罠を完全にシャットアウトするためのウチの修復プラン（防衛ライン）は2つだし！
まず、データが増殖するのを防ぐために、合体させる前にコンテナB（売上履歴）の重複を `groupby()` でギュッと1行に集約（集計）して、ユーザーごとの「合計金額」のきれいな表に整えてから合体させる！
そして、佐藤さんのところに発生してしまった「NaN（空っぽのゴミ）」は、放置すると計算エラーのバグになるから、`fillna(0)` を使って「購入金額：0円」として美しく安全に埋める処理を強制適用するだし！

これならデータが化ける心配は一切ないし、ナノのイチャモンも完全シャットアウトだし！ステラ、この通りのロジックでコードを安全に昇華しなさい！
</spica_fix>

<stella_code>
```python
import pandas as pd
import traceback

def merge_and_optimize_customer_sales(df_customers: pd.DataFrame, df_sales: pd.DataFrame) -> pd.DataFrame:
    try:
        # 🦔 スピカの防衛要求に従い、まずは売上履歴の重複をユーザーIDごとに合計集計だし！
        # ⚡ ナノが指摘した「U001（田中さん）のデータ増殖トラップ」をこれで事前に完全破壊するよ！
        df_sales_grouped = df_sales.groupby('ユーザーID', as_index=False)['購入金額'].sum()
        
        # 🤝 2つのきれいなコンテナを「ユーザーID」をキーにして安全に合体（左結合）
        df_merged = pd.merge(df_customers, df_sales_grouped, on='ユーザーID', how='left')
        
        # 🛡️ ナノが指摘した「売上がない佐藤さんのNaNバグ」を、スピカの指示通り0円で美しく穴埋め！
        df_merged['購入金額'] = df_merged['購入金額'].fillna(0)
        
        # 📈 最後に購入金額の型をきれいな整数型（int64）に整えて、CODE_STABLE（完全体データ）の完成！
        df_merged['購入金額'] = df_merged['購入金額'].astype('int64')
        
        return df_merged
        
    except Exception as e:
        print(f"❌ ARCHITECTURE_ERROR_LOG:\n{traceback.format_exc()}")
        raise e
